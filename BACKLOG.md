# Backlog（優先度順ではなく発生順のメモ）

## セッションの永続化 / 再接続（アプリ再起動後も前回セッションを自動再開）
- 要望: アプリを終了→再起動しても、各カードが前回のターミナル/Agentセッションを自動的に選んで起動する。
- アプローチ案:
  - **Claude 前提なら実現性高**: カード初回起動を `claude --continue`（その cwd の直近セッションを再開）にする。
    確実にしたいならカードに最後の `session_id` を保存し `claude --resume <session-id>`。
    - Claude のセッションは `~/.claude/projects/<project>/<session-id>.jsonl` に保存される。
  - 汎用シェル/プロセスの“プロセスそのもの”を再起動跨ぎで生かすのは tmux 的デタッチが必要で重い。
    まずは Claude の `--continue` で実用的に解決するのが現実的。
- 関連: スライス3（Agent状態検出）の「バックグラウンド PTY」設計と重なるので一緒に検討。
